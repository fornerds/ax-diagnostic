# 키 발급 가이드 — 고도몰 (godoshopping)

> 본 가이드는 검증본(`_workspace/godoshopping/02_validation.md`)과 `README.md`에서 출처가 확인된 사실만 담는다. 추측 금지. 발급 콘솔의 일부 화면 경로는 개발자센터 페이지가 JS 렌더(본문 미수집)라 본 가이드 작성 시점 공식 본문으로 단정할 수 없어 "미확인"으로 표기했다.

## 연동 유형 / 인증 방식

- **연동 방식**: POST 웹통신, 응답 데이터는 **XML(UTF-8)** (JSON 아님). (출처: specDownload p.11 "1.2 / 1.3 / 1.4")
- **인증 방식**: **API Key 이중키**. 모든 API 요청 본문에 `partner_key`(제휴사 인증키) + `key`(쇼핑몰 인증키) 둘 다 필수 동봉. (출처: specDownload p.12 "1.7 API 요청시 기본 데이터")
- **OAuth/토큰 흐름 없음**: 토큰 교환·리프레시 단계 없는 정적 키. 키는 사전 신청·승인으로 발급. (출처: specDownload p.11 "1.5")
- sandbox 키로 테스트 → real 키로 운영 전환.

## 발급 절차 (스텝)

발급 흐름 (1차 출처: specDownload p.11 "1.5" / 02_validation 리스크 A; 발급 위치는 NHN커머스 개발자센터 도움말로 확인 — 아래 "출처" 참조):

1. **NHN커머스 회원가입** 후 개발자센터(`https://devcenter.godo.co.kr`) 접속·로그인.
2. **제휴사키(`partner_key`) 신청 — "오픈 API → 키발급 신청".**
   - 메뉴 경로: 개발자센터 화면 **상단 우측 [오픈 API]** 클릭 → **좌측 메뉴 [키발급 신청]** → **[키 발급 신청 폼]** 입력 → **[신청] 버튼** 클릭. (자체개발 트랙은 "오픈 API → 개발자 등록"으로도 진입.) 신청 후 **승인 대기**.
   - 직링크: 개발자센터의 키발급/키관리 콘솔 URL은 **제휴사(에이전시) 슬러그가 경로에 포함되는 워크스페이스별 주소**(예: `https://devcenter.nhn-commerce.com/{제휴사슬러그}/openapi/keyLicense`)라서 **모든 계정에 공통으로 쓰는 범용 직링크는 없음.** 위 메뉴 경로로 진입할 것.
3. **사용자키(쇼핑몰 인증키 `key`) 신청·승인.**
   - 제휴사키 승인 후, 각 제휴사용으로 생성된 **사용자키 신청 URL**(개발자센터 **마이페이지 → 사용자키 신청**에서 확인)에서 연동할 쇼핑몰을 선택해 신청.
   - 신청 시 해당 쇼핑몰을 보유한 고도 회원 아이디 이메일로 **최종 확인 메일** 발송 → 유효기간 내 메일 확인·사용자 인증 완료 시 신청 완료. 발급된 key는 개발사에 전달되고, 쇼핑몰 보유 회원에게는 발급 완료 내역이 SMS로 통지.
   - 직링크: 사용자키 신청 URL도 **제휴사 슬러그 포함 워크스페이스별 주소**(예: `.../{제휴사슬러그}/openapi/userKey`)라 범용 직링크 없음. 마이페이지에서 확인.
4. **테스트는 sandbox 키 → 정식 운영은 real 키**로 전환.

- 발급 소요: 통상 **2~3 영업일**(보조출처: bizesm FAQ — 02_validation 리스크 A).
- 화면 내 정확한 버튼 라벨은 트랙·테마에 따라 "신청"/"등록하기" 등으로 표기될 수 있음(개발자센터 도움말 기준 [신청]). 모든 인증키 신청은 **개발자센터를 통해서만** 가능.

## 전제조건

- **사업자 인증 필요**: 누구나 즉시 호출 불가. 사업자 인증 + API 사용 신청 승인이 전제. (출처: specDownload p.11 "1.5"; 02_validation 리스크 A)
- **심사/승인 대기**: 개발자 등록·API 사용 신청 모두 승인 절차가 있음(통상 2~3 영업일).
- **발신번호 사전등록**: 해당 없음(본 API는 SMS/알림 발송 API가 아님) — **없음**.
- **플랜 요건**: 본문에 명시된 별도 유료 플랜 요건 **미확인**(스펙·README에 플랜 등급 요건 기재 없음).
- **IP 화이트리스트**: 허용 IP 외 호출 시 에러코드 996 발생(메커니즘 존재 확인). 단, **등록이 모든 계정에 강제인지·등록 위치는 공식 본문 미기재 → 미확인**. 런타임에 996 수신 시 "허용 IP 등록 필요"로 안내. (출처: specDownload p.13 "996"; 02_validation 리스크 E)

## 스코프 / 권한

- **명시적 스코프 체계 없음**(OAuth 스코프 아님). 접근 범위는 **키 타입에 종속**. (출처: 02_validation 인증 확정 / 리스크 B)
- **본사 인증키 vs 공급사 인증키**로 사용 가능 API·범위가 다름:
  - 공급사 인증키는 게시판 일부(**상품후기·상품문의·공급사 게시판**)만 사용 가능. (출처: specDownload p.69 "5.1", p.77 "5.7")
  - 공급사 인증키는 주문/상품 일부 항목 설정·수정 불가(상품명·판매가·판매기간 등). (출처: specDownload p.29 "3.6", p.38 "3.7", 코드 401)
- 키 타입을 사전 판별하는 API는 스펙에 없음 → 권한 부족 시 도구가 에러코드(401/403/995/996)를 그대로 노출. (출처: 02_validation 리스크 B)

## 환경변수 매핑

| 변수 | 값을 어디서 | 필수 |
|------|------------|------|
| `GODO_PARTNER_KEY` | 제휴사 등록 시 발급된 **제휴사 인증키(partner_key)** — 개발자센터 키 관리 | 예 |
| `GODO_KEY` | 쇼핑몰 API 사용 승인 시 발급된 **쇼핑몰 인증키(key)** — 개발자센터 키 관리 | 예 |
| `GODO_ENV` | `real`(기본, https) 또는 `sandbox`(http 평문, 테스트 전용) 전환 값 | 아니오 |
| `GODO_DATA_URL_BASE` | `data_url` 방식 도구(goods_insert/goods_update/goods_stock_update)에서 서버가 호스팅할 임시 XML의 공개 베이스 URL | 조건부 |

> 시크릿은 코드나 `.mcp.json`에 실값으로 쓰지 말고 환경변수 / `${VAR}` 확장으로 주입한다.

베이스 URL (출처: specDownload p.11 "1.6"):
- real: `https://openhub.godo.co.kr/godomall5/{group}/{Action}.php`
- sandbox: `http://sbopenhub.godo.co.kr/godomall5/{group}/{Action}.php` (http 평문, 테스트 한정)

## 미확인 / 주의

- **콘솔 내 키 발급·확인 화면의 정확한 메뉴 경로**: 확인됨 — 개발자센터 상단 [오픈 API] → 좌측 [키발급 신청] → [신청](제휴사키), 마이페이지 → 사용자키 신청(사용자키). 단 **키발급/사용자키 신청의 직링크 URL은 제휴사 슬러그가 포함된 워크스페이스별 주소라 범용 직링크는 부재**(위 "발급 절차" 참조). (출처: devcenter-help.nhn-commerce.com 도움말)
- **IP 화이트리스트 강제 여부·등록 위치**: 미확인 (코드 996만 확인). 996 수신 시 허용 IP 등록 필요로 안내.
- **유료 플랜 등급 요건**: 미확인 (스펙·README에 기재 없음).
- **혼동 주의**: 본 대상은 **godomall5(고도몰5) Open API**. NHN커머스의 별도 플랫폼 **shopby(샵바이)** API와 다른 제품이므로 shopby 문서/엔드포인트를 끌어다 쓰지 말 것. (출처: 02_validation 리스크 F)
- **회원(고객) 전용 조회 API 부재**: 회원 데이터는 `order_search` 응답(memNo/memId/memGroupNm)로 파생. (출처: 02_validation 리스크 C)
