# 키 발급 가이드 — 두레이(Dooray!) (dooray)

> 출처 기준일: 02_validation.md 검증일 2026-06-26 (1차 출처: Dooray 서비스 API 레퍼런스, 문서 표기 "Update 2026.06.11")

## 연동 유형 / 인증 방식
- **연동 유형**: 로컬 stdio MCP 서버 (공식 원격 MCP 커넥터 없음 → "로그인 연결" 불가, 개인 인증 토큰을 env로 주입하는 경로가 유일).
- **인증 방식**: HTTP 헤더 `Authorization: dooray-api {TOKEN}` (TOKEN 값만, `{}` 미포함). 정적 개인 인증 토큰 사용. **OAuth 미사용**, 토큰 교환/리프레시 흐름 없음.
- **공통 규약**: JSON 바디 전송 시 `Content-Type: application/json` 필수. TLS 1.2+ 강제(1.0/1.1 미지원).

## 발급 절차 (스텝)
1. 본인 Dooray 계정으로 웹 로그인.
2. **개인설정 > API > 개인 인증 토큰** 메뉴로 이동.
3. 개인 인증 토큰을 생성(자가발급)한다. 심사/제휴 절차 없음.
4. 생성된 토큰 값을 복사해 환경변수 `DOORAY_API_TOKEN` 에 주입한다.
5. (선택) 테넌트 환경이 민간이 아니면 `DOORAY_BASE_URL` 을 해당 환경 도메인으로 설정한다(아래 표 참고).

> 출처: 서비스 API 레퍼런스 §인증 > 개인 API 인증 토큰 발급 과정 — "개인설정 > API > 개인 인증 토큰 메뉴에서 생성합니다" (검증됨).

## 전제조건
- **플랜/사업자 심사/제휴 심사**: 없음 (설정 메뉴에서 자가발급, 문서상 심사 절차 부재 — 검증됨).
- **발신번호 사전등록 등**: 해당 없음 (서비스 API).
- **네트워크 제약(주의)**: 공공/공공업무망/금융 테넌트는 **IP ACL** 이 걸린 경우가 많다. MCP 서버가 도는 네트워크가 허용 IP 여야 호출이 성공한다(코드로 우회 불가 — 네트워크 제약).

## 스코프 / 권한
- **스코프 개념 없음**. 토큰 = **발급 계정과 동일한 권한**.
- 발급 계정에 적용되는 **IP ACL / User ACL 이 토큰에도 동일 적용**된다.
- 따라서 MCP가 다룰 수 있는 범위는 토큰 주인의 권한에 종속된다.

> 출처: 서비스 API 레퍼런스 §인증 > 개인 API 인증 토큰 권한 — "발급 받은 계정과 동일한 권한", "ACL(IP ACL, User ACL) 동일 적용" (검증됨).

## 환경변수 매핑

| 변수 | 값을 어디서 | 필수 |
|------|------------|------|
| `DOORAY_API_TOKEN` | Dooray 웹 > 개인설정 > API > 개인 인증 토큰 에서 자가발급한 토큰 값. 헤더 `Authorization: dooray-api {값}` 으로 전송 | **필수** |
| `DOORAY_BASE_URL` | 테넌트 환경 도메인(아래 환경별 표). 미설정 시 민간 기본값 사용 | 선택(기본 `https://api.dooray.com`) |

**환경별 베이스 URL** (전 항목 검증됨, 서비스 API 레퍼런스 §기본 > End Point)

| 환경 | `DOORAY_BASE_URL` |
|------|-------------------|
| 민간(기본) | `https://api.dooray.com` |
| 공공 | `https://api.gov-dooray.com` |
| 공공업무망 | `https://api.gov-dooray.co.kr` |
| 금융 | `https://api.dooray.co.kr` |

## 미확인 / 주의
- **개인 토큰 만료/회전 정책**: 생성/폐기는 문서 확인되나 **자동 만료 정책은 본문에 명시 없음(미확인)**. 401(토큰 없음·폐기·오류) 발생 시 재발급으로 대응한다.
- **관리 API 혼동 금지**: `admin-api.*` / `/admin/v1/...` (온보딩팀 수동 발급 토큰 필요한 테넌트 관리용)은 **이 커넥터 대상 아님**. 반드시 서비스 API(`api.dooray.com`, 개인 토큰)만 사용한다.
- **OAuth/사이드앱 인증 플로우 상세**: API 허브에 링크는 존재하나 본 MCP는 개인 토큰으로 충분 → **미열람(미확인)**. 본 커넥터는 개인 토큰만 사용.
- **공공/금융 클라우드의 서비스별 API 가용 차이**: 환경별 제약 문서화 부재(미확인). 베이스 URL env + 런타임 에러 처리로 대응.
